Session ERD

一份会话系统的数据模型草图：从 USERS 到 CONVERSATIONS 到 MESSAGES， 再到压缩出的 SESSION_CONTEXT、长期 USER_MEMORIES，以及 RAG 用的 DOCUMENT_CHUNKS。 可作为对话型 AI 系统持久层的参考起点。

关系图

下图用 Mermaid 渲染。展开主键、外键、关键字段；省略了审计字段与索引细节。

设计要点

  • 会话即聚合根 — CONVERSATIONS 持有 token、消息数与最后活动时间，便于配额、计费与超时治理。
  • 消息是事实表 — MESSAGES 用 seq_no 严格有序，content_type 区分文本 / 工具调用 / 工具返回；input_tokens / output_tokens / latency_ms 作为可观测性的内嵌指标。
  • 上下文压缩单独建表 — SESSION_CONTEXT 记录被压缩的消息区间与总结文本，并用 is_active 标记当前生效的那条；多份历史快照可保留以便回放。
  • 记忆是用户级别的 — USER_MEMORIES 与会话解耦，按 memory_type（user / feedback / project / reference 等）分类；embedding 与 importance 用于检索与衰减。
  • RAG 与会话分离 — DOCUMENTS / DOCUMENT_CHUNKS 是知识库侧的实体，按 kb_version 管理 reindex，避免污染会话语义。

取舍

把"上下文压缩"和"用户记忆"分开两张表，是因为两者的生命周期与访问模式完全不同：压缩是会话内部的实现细节，频繁写、按会话读；用户记忆是跨会话的、按相似度检索的、生命周期更长（甚至跨产品）。合并到一张表会让查询路径变复杂、索引选择两难。
